## Directive Selection: UseRoutes=no vs UseGateway=no

During development, `UseGateway=no` was evaluated as a narrower alternative to
`UseRoutes=no` (suppresses only the default gateway route, not classless static routes).
However, live testing on the target stemcell revealed that `UseGateway=` was introduced in
**systemd 250**, while Ubuntu 22.04 Jammy ships **systemd 249**.

The journal log captured during the live test confirmed this:

```
systemd-networkd[886]: /etc/systemd/network/10_eth1.network:12:
  Unknown key name 'UseGateway' in section 'DHCP', ignoring.
```

`UseRoutes=no` has been available since systemd 217 and is fully supported on all Ubuntu
Jammy stemcells. It prevents both the default gateway route and any classless static routes
(DHCP option 121) from being installed by the secondary NIC. For a secondary DHCP NIC that
shares the same subnet as the primary NIC, all DHCP-learned routes would be duplicates
anyway, so the broader suppression is correct behavior.

## Prior Art

This fix closes a gap that existed for DHCP networks only.  Static networks
(`type: manual`) were already handled correctly: the `UbuntuNetManager` uses
`routeType` to detect whether a static NIC is the default gateway, and only
writes a default route for that NIC.  DHCP networks lacked the equivalent
`IsDefaultForGateway` flag on `DHCPInterfaceConfiguration`, so every DHCP NIC
received identical `[DHCP]` sections and every one installed a default route via
`systemd-networkd`'s `UseRoutes` default of `yes`.
